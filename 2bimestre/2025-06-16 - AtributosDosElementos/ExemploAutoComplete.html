Exemplo de Autocomplete em Inputs HTML

Experimente preencher os campos abaixo. Seu navegador pode oferecer sugestões de preenchimento automático com base no histórico.

Campos com Autocomplete Habilitado (`autocomplete="on"`)

Nesses campos, o navegador é incentivado a oferecer sugestões de preenchimento.

`autocomplete="name"` é uma dica específica para nomes.


`autocomplete="email"` é uma dica específica para endereços de e-mail.


`autocomplete="tel"` é uma dica específica para números de telefone.


`autocomplete="street-address"` é uma dica para endereços de rua.


`autocomplete="address-level2"` é uma dica para a cidade.

Campo com Autocomplete Desabilitado (`autocomplete="off"`)

Nesse campo, o navegador é instruído a *não* oferecer sugestões de preenchimento.

É comum desabilitar o autocomplete para campos de senha por segurança ou para campos que não devem ter histórico salvo.


Quando não especificado, o padrão geralmente é `on`. Ou, se você especificar `on`, o navegador tentará autocompletar com base no nome do campo ou no histórico geral.


Mesmo em campos genéricos, você pode forçar o `off`.

Observação Importante: Embora `autocomplete="off"` seja uma instrução para o navegador, ele **não é uma garantia de segurança**. Alguns gerenciadores de senha e navegadores podem ignorar essa instrução por motivos de usabilidade ou segurança do usuário (por exemplo, para permitir que o usuário use seu gerenciador de senhas integrado). Para dados altamente sensíveis, é crucial implementar validações e controles de segurança no lado do servidor.
